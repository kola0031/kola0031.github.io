Layout

Breakpoints

Bootstrap breakpoints lets you create customizable widths that helps with responsive layout on all devices or screen sizes

Core Concepts
  • Breakpoints are the pillars of responsive design.They can be used to control layout adaptability at a particular viewport or screen sixe.
  • You can use media queries to structure your CSs by breakpoint. Media Query is a core part of CSS that allows users to unconditionally apply styles based on a set of browser and operating system parameters
  • Mobile first, responsive design is the Goal.
Available breakpoints

Bootstrap comes with sex default breakpoints, usually referred as grid tiers for building responsively. These breakpoints can be customized by using Booutstrap SASS files

Breakpoints are selected to comfortably hold containers whose widths are multiples of 12. They are also representative of a subset of common device sizes and viewport dimensions; they do not specifically target every use case or device.

Containers

Containers are a very significant tool of Bootstrap that contain, pad, and align content within a given device or screen

How they work

Containers are the most basic layout element in Bootstrap and is important when using Bootstraps default grid system

You can use containers to contain, pad, and (sometimes) center the content within them. Most layouts can be nested but it is not a requirement.

Bootstrap comes with three different container classes:

  • .container(This sets a max-width at each responsive breakpoint)
  • .container-fluid (This sets a width of 100% at all breakpoints)
  • .container-{Breakpoint} (This sets a width of a 100% at the specified breakpoint)

Bootstrap's default container class (.container) is a responsive, fixed width container, meaning that its max-width is going ti change at each breakpoint

  • <div class="container">
Responsive containers

We also have responsive containers which allow you to specify a class that is 100% wide until it reaches a certain breakpoint. Max-width is applied when max breakpoint is reached. For example .container-sm is 100% wide to start until the sm breakpoint is reached where it will scale up with md, lg, xl, and xxl

For Example:

  • <div class="container-sm">100% wide until small breakpoint</div>
  • <div class="container-md">100% wide until meduium breakpoint</div>
  • <div class="container-lg">100% wide until large breakpoint</div>
  • <div class="container-xl">100% wide until extra large breakpoint</div>
  • <div class="container-xxl">100% wide until extra extra large breakpoint</div>
Fluid containers

You use .container-fluid when you want a full width container that spans the entire width of the viewport

For Example:

  • <div class="container-fluid">

The below tags should be used for semantic purpose

  • <mark>represents text which is marked or highlighted for reference or notational purpose
  • <small> represents side-comments and small print, like copyright and legal text.
  • <s>represents elements that are no longer relevant or no longer accurate
  • <u>represents a span of inline text which should be rendered in a way that indicates that it has a non-textual annotation.

But you can use the following classes to style your text

  • .mark will apply the same styles as <mark>
  • .small will apply the same styles as <small>
  • .text-decoration-underline will apply the same styles as <u>
  • .text-decoration-line-through will apply yhr same styles as <s>.
